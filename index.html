安装过程

Windows直接下载Composer-Setup.exe运行就好了，自动安装多方便~

Linux下，全局安装用下面两条命令：

curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

OK~安装就这么简单，然后来看看怎么使用composer管理项目依赖

命令行进入项目所在目录，运行命令：

composer install

这时候你会发现install的非常非常的慢，没办法因为墙太高了嘛，所以我们要用到镜像

有两种方法可以使用国内的镜像，一个是全局配置，进入命令行输入：

composer config -g repo.packagist composer https://packagist.phpcomposer.com

一个是在你的项目里配置，在项目目录下命令行输入：

composer config repo.packagist composer https://packagist.phpcomposer.com

如何在Cpython调用自己写在文件里的函数

　　这个问题好像很小白- -但是谁还不是从小白过来的呢~

  • 首先cmd命令行进入你py文件所在的目录
  • 输入python进入交互环境，输入 import afunc (如果 afunc.py 是你的文件全名的话）

    import afunc

  • 接下来就可以调用你写在这个文件里的函数了

    afunc.函数名(参数)

一条神奇的语句a,b=b,a+b

　　不知道你看着懵逼不，反正第一次见到它我是懵逼了，这是撒？逗号表达式嘛，显然不是啊。研究了半天发现其实这条语句是这样执行的(a=1,b=2)：

>>>t=b,a+b
>>>t
(2,3)
>>>a,b=t
>>>a
2
>>>b
3

　　也就是说在a,b=b,a+b这个表达式中，首先将等号右边赋值给一个tuple元组，然后a,b分别被赋值了这个元组的相应元素。感觉就像是(a,b)=(b,a+b)。至于为什么是tuple而不是list，我想是为了保护数据不被修改吧，因为tuple规定了一旦赋值就不可修改~

终于搭好的Github+hexo Blog

发表于 2017-03-28 | 阅读次数

GitHub注册也有一两年了，一直只用来clone代码，从来都没好好使用说起来都觉得惭愧了- - 今天折腾了半天的时间，总算是把博客搭起来了，以后那些无聊的碎碎念和学习日报什么的终于有地方放啦~之前wordpress那个博客反正也没有太多东西，就不备份上来，就当做重新开始，重新做人哈哈~fighting~

搭建过程

本地Hexo搭建

  1. 安装Git
  2. 安装Node.js
  3. 安装Hexo
    npm install -g hexo
  4. 选择一个本地目录作为你的博客本地存放目录，例如D:\hexo
  5. 进入目录下打开git bash
    hexo init
npm install 
npm install hexo -server --save
  6. 这样本地的博客就搭好了，输下面的命令启动,然后浏览器打开localhost:4000
    hexo g
hexo s

部署到Github

  1. 注册github，不细写啦
  2. 创建新仓库，用做博客的页面仓库一般都是yourname.github.io这样的格式

  3. 生成SSH密钥，这项可选。

    ssh-keygen -t rsa -C "注册账号时用的邮箱地址"

    都按回车就好。这样在C:\Users\Administrator.ssh下就生成了两个文件id_rsa和id_rsa.pub。打开id_rsa.pub全部复制，然后打开你的github的settings页面，Add SSH key，把刚刚复制的粘贴进去就好了。

  4. 修改hexo目录下的_config.yml配置文件，把文件拉到最后，改下面这些内容：
    deploy:
type: git
repository: git@github.com:ChenYuPiao/ChenYuPiao.github.io.git
branch: master

  5. 创建文章可用以下命令，然后进入scource_posts目录修改md文件

    hexo new “tittle”

    注：这里””里面最好不要用中文，容易出现各种错误啊别问我咋知道的- -标题可以打开md文件从里面修改。hexo的其他命令也不细说了，网上很多啦。

  6. 最后一步，用下列命令部署

    hexo clean 这个其实可以省略，但是为了避免错误，你懂得，最好在提交之前用这个命令清下缓存
hexo g
hexo d

    运行后看到[info] Deploy done: git 就说明成功了~打开浏览器输入yourname.github.io试试吧~

独立域名

很多人可能觉得github的二级域名不够拉风，可以去万网或者godaddy注册一个域名，费用嘛一年五六十到五六块不等~

  1. 设置域名解析，一般购买域名的网站上都可以进行域名管理，找到域名解析，添加两个解析

    CNAME www yourname.github.io
    A @ IP //命令行ping一下yourname.github.io就能看到IP

  2. 设置DNS地址，可以用默认的，也可以改成DNSpod的
  3. 在本地source下添加一个CNAME文件，无后缀类型，里面写上你刚注册的域名。

没有那么快生效，可能要等到第二天。

更改Hexo主题

官网https://hexo.io/themes/ 有很多好看的主题，可以自行挑选~

  1. 下载主题
    git clone https://github.com/wuchong/jacman.git themes/jacman  //网址自己clone替换
  2. 修改你的博客根目录下的config.yml配置文件中的theme属性，将其设置为jacman（主题名称）
  3. 重复上面部署的过程，重新启动。
  4. 主题目录下的config.yml也可以修改，用来定制你自己的网站，一般主题都会有修改说明。
