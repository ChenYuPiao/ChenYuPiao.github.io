php包管理器composer

发表于 2017-04-12 | 阅读次数 |

接触php也有这么久了，一直只是改改代码写写页面，开始做项目了才知道php也有依赖管理工具啊！！跟Node的npm一样~

安装过程

Windows直接下载Composer-Setup.exe运行就好了，自动安装多方便~

Linux下，全局安装用下面两条命令：

curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

OK~安装就这么简单，然后来看看怎么使用composer管理项目依赖

使用tips

首先要先在项目目录下建立一个composer.json的文件

这个文件里就写了你整个项目需要用到的依赖，比如说像下面这样写：

主要就是写清楚你要用到的包名称和包版本。

命令行进入项目所在目录，运行命令：

composer install

这时候你会发现install的非常非常的慢，没办法因为墙太高了嘛，所以我们要用到镜像

有两种方法可以使用国内的镜像，一个是全局配置，进入命令行输入：

composer config -g repo.packagist composer https://packagist.phpcomposer.com

一个是在你的项目里配置，在项目目录下命令行输入：

composer config repo.packagist composer https://packagist.phpcomposer.com

依赖装好了怎么使用呢，就需要用到下面这条语句：

require ‘vendor/autoload.php’;
或者在composer.json中配置自己的autoloader
